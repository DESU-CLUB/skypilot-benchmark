**Key design points:**
- `GIT_TOKEN` is declared in `envs` with placeholder `your-token-here` — override it at launch time with `sky launch --env GIT_TOKEN=<real-token>` so the actual secret never lives in the YAML.
- The `setup` command injects the token into the HTTPS clone URL via `${GIT_TOKEN}`, which is the standard credential-less approach for ephemeral cloud VMs.
- No GPU is requested — `n1-standard-4` is a 4-vCPU / 15 GB RAM CPU-only GCP instance type.
