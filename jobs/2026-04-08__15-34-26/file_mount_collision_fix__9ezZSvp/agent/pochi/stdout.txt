### Root cause

SkyPilot applies `file_mounts` **before** `setup` runs. In `broken_task.yaml` the GCS bucket was mounted at:

```
/home/user/sky_workdir/my-repo   ← file_mounts target
```

Then `setup` tried:

```bash
git clone https://github.com/my-org/my-repo.git /home/user/sky_workdir/my-repo
```

Because the directory already existed and was non-empty (populated by the mount), `git` exited 128: `fatal: destination path already exists and is not an empty directory.`

---

### Fix applied (`/home/user/fixed_task.yaml`)

The `file_mounts` target was renamed to a path that does **not** overlap with the clone destination:

```yaml
file_mounts:
  /home/user/sky_workdir/my-repo-configs:   # ← was /home/user/sky_workdir/my-repo
    source: gs://my-configs/repo-configs
    mode: MOUNT
```

The `setup` and `run` sections are unchanged — they still clone into and reference `/home/user/sky_workdir/my-repo`, which is now unoccupied when `setup` runs.
